## memory store location

- Existing persona artifacts are run-scoped. `orchestrator/run.py:120` creates `artifacts/<run_id>`, and `build_persona_config` derives `artifact_dir` as `run_dir / "personas" / template.persona_id` at `orchestrator/run.py:112`. `run_persona` writes transcript, screenshots, decisions, and bug reports inside that run artifact directory at `agents/personas/run.py:252` through `agents/personas/run.py:254`.
- That makes `artifacts/<run_id>/...` the wrong home for persistent persona memory. It is useful evidence for one run, but not stable across run ids.
- There is currently no persistent persona-memory implementation. A repo search for memory-related storage found only the app's reset response strings `in_memory_cart` at `app/main.py:282` and `app/buggy_main_seed.py:282`, not a persona memory store.
- Recommended stable location: `memory/personas/<persona_id>.json`. It is run-independent, keyed by the stable `persona_id` from `configs/run_config.json`, and separated from generated artifacts.
- Because `PersonaTemplateV1` forbids extra config fields at `shared/contracts/models.py:12` through `shared/contracts/models.py:13` and only declares the fields at `shared/contracts/models.py:169` through `shared/contracts/models.py:175`, do not add `memory_path` to `configs/run_config.json` unless the contract is intentionally changed. The safer first implementation is to derive the path from `persona_id`.
